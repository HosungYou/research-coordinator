## 개요

### Diverga란?

Diverga는 **37개 전문 연구 에이전트**를 조율하여 연구의 전체 생애주기를 지원하는 AI 연구 어시스턴트입니다.

### 핵심 차별점

| 기존 AI 도구 | Diverga |
|-------------|---------|
| 단일 에이전트 | **37개 전문 에이전트** |
| 항상 같은 답변 (모드 붕괴) | **VS 방법론으로 창의적 대안 제시** |
| AI가 모든 결정 | **인간 체크포인트에서 연구자가 결정** |
| 순차 처리만 | **병렬 실행으로 5배 빠른 처리** |

### 핵심 원칙

```
"인간이 할 일은 인간이, AI는 인간의 범주를 벗어난 것을 수행"
```

## 오케스트레이션 메커니즘

### 메커니즘 1: 순차 체이닝

에이전트 출력이 다음 에이전트의 입력이 되는 방식

```
A1 ────► A2 ────► C1 ────► 결과
  연구질문   이론구축   양적설계
```

**사용 시점**: 각 단계가 이전 단계의 결과에 의존할 때

**예시**:
```yaml
단계1: A1-연구질문정제기
  입력: "AI가 학습에 미치는 효과"
  출력: PICO 형식의 연구 질문

단계2: A2-이론프레임워크아키텍트
  입력: 정제된 연구 질문
  출력: 이론적 프레임워크

단계3: C1-양적설계컨설턴트
  입력: 이론적 프레임워크
  출력: 연구 설계
```

### 메커니즘 2: 병렬 실행

여러 에이전트가 동시에 독립적으로 작업

```
         ┌─── B1 문헌검색 ───┐
입력 ────┼─── B2 품질평가 ───┼──► 통합 결과
         └─── B3 효과크기 ───┘
```

**사용 시점**: 에이전트들이 서로 독립적으로 작업할 수 있을 때

**구현 방법** (Claude Code에서):
```xml
<!-- 단일 메시지에 여러 Task = 병렬 실행 -->
<function_calls>
  <invoke name="Task">B1 문헌 검색</invoke>
  <invoke name="Task">B2 품질 평가 준비</invoke>  ← 동시 실행!
  <invoke name="Task">B3 효과크기 템플릿</invoke>
</function_calls>
```

### 메커니즘 3: 코디네이터-워커 패턴 (B5)

대규모 작업을 분할하여 여러 워커가 병렬 처리

```
              ┌─────────────────┐
              │   COORDINATOR   │  (Opus)
              │    코디네이터    │
              │                 │
              │ • 작업 분할      │
              │ • 워커 생성      │
              │ • 결과 집계      │
              └────────┬────────┘
                       │
       ┌───────────────┼───────────────┐
       ▼               ▼               ▼
 ┌───────────┐  ┌───────────┐  ┌───────────┐
 │ Worker 1  │  │ Worker 2  │  │ Worker N  │  (Haiku)
 │ PDF 1-20  │  │ PDF 21-40 │  │ PDF ...   │
 └─────┬─────┘  └─────┬─────┘  └─────┬─────┘
       │              │              │
       └──────────────┼──────────────┘
                      ▼
                 통합 결과
```

**B5의 핵심 장점**:
- 메모리 오버플로우 방지 (각 워커가 독립적 컨텍스트 사용)
- 5배 빠른 처리 (순차 vs 병렬)
- 내결함성 (워커 1 실패해도 나머지는 계속 진행)

### 메커니즘 4: 휴먼 체크포인트

중요한 결정 지점에서 AI가 멈추고 인간의 승인을 기다림

```
에이전트 작업 ───► 🔴 체크포인트 ───► 인간 결정 ───► 계속 진행
```

| 체크포인트 유형 | 아이콘 | 동작 |
|----------------|--------|------|
| **필수** | 🔴 | 시스템 정지 - 승인 없이 진행 불가 |
| **권장** | 🟠 | 시스템 일시정지 - 승인 강력 권장 |
| **선택** | 🟡 | 시스템 질의 - 생략 시 기본값 사용 |

**주요 체크포인트**:
- `CP_RESEARCH_DIRECTION` - 연구 질문 확정
- `CP_PARADIGM_SELECTION` - 양적/질적/혼합 선택
- `CP_THEORY_SELECTION` - 이론적 프레임워크 선택
- `CP_METHODOLOGY_APPROVAL` - 연구 설계 승인

## 워크플로우 예시

### 예시 1: 체계적 문헌고찰 단계별 실행

```yaml
상황: "AI 챗봇이 언어 학습에 미치는 효과에 대한 메타분석"

=== 1단계: 연구 질문 정제 ===

사용자 입력: "AI 챗봇이 언어 학습에 미치는 효과를 체계적으로 검토하고 싶어요"

에이전트: A1-연구질문정제기
실행:
  - PICO 프레임워크 적용
  - P (대상): 언어 학습자 (K-12, 대학생)
  - I (중재): AI 챗봇 중재
  - C (비교): 전통적 교수, 무중재
  - O (결과): 말하기 능숙도, 동기, 불안

출력: "AI 챗봇 기반 언어 학습 중재가 학습자의 말하기 능숙도에 미치는 효과는 어떠한가?"

🔴 체크포인트: CP_RESEARCH_DIRECTION
사용자 결정: "말하기 능력에 초점을 맞추어 승인"

=== 2단계: 이론적 프레임워크 ===

에이전트: A2-이론아키텍트
실행:
  - VS 방법론 적용
  - 모달 경고: "TAM (T=0.92)은 너무 뻔한 선택"

출력 (대안 제시):
  ├─ 방향 A (T≈0.55): 사회문화이론 + TAM
  ├─ 방향 B (T≈0.38): 인지부하이론 + 자기결정이론 ⭐ 권장
  └─ 방향 C (T≈0.22): 분산인지 + 플로우 이론

🔴 체크포인트: CP_THEORY_SELECTION
사용자 결정: "방향 B 선택"

=== 3단계: 문헌 검색 ===

[병렬 실행]
  ├─ B1-문헌검색전략가: PRISMA 2020 검색 전략 수립
  │   데이터베이스: Semantic Scholar, OpenAlex, PubMed, ERIC, PsycINFO
  │
  └─ B4-연구레이더: 인용 알림 설정

출력:
  - 식별된 레코드: 2,847개
  - 중복 제거 후: 1,923개

=== 4단계: PDF 병렬 처리 ===

에이전트: B5-병렬문서처리기

구성:
  - 워커 수: 7
  - 배치 크기: 워커당 15개
  - 추출 유형: 전체 텍스트 + 메타데이터

실행 과정:
  ┌──────────────────────────────────────┐
  │ 코디네이터 (Opus): 작업 분배         │
  │                                      │
  │ Worker 1: PDF 1-19   ████████░░ 80%  │
  │ Worker 2: PDF 20-38  ██████░░░░ 60%  │
  │ Worker 3: PDF 39-57  ██████████ 100% │
  │ Worker 4: PDF 58-76  ████░░░░░░ 40%  │
  │ Worker 5: PDF 77-95  ████████░░ 80%  │
  │ Worker 6: PDF 96-114 ██████░░░░ 60%  │
  │ Worker 7: PDF 115-127 ██████████ 100%│
  │                                      │
  │ 전체 진행률: 89/127 (70%)            │
  └──────────────────────────────────────┘

출력:
  - 총 처리: 127개 (스크리닝 후)
  - 성공: 124개
  - 실패: 3개 (OCR 필요)

=== 5단계: 품질 평가 & 효과크기 추출 ===

[병렬 실행]
  ├─ B2-근거품질평가자: RoB 2.0 평가
  │   출력: 고품질 연구 89개
  │
  └─ B3-효과크기추출기: Cohen's d, Hedge's g 추출
      출력: 효과크기 156개 추출

=== 6단계: 메타분석 ===

에이전트: E1-양적분석가이드
실행: 무선효과 메타분석
출력:
  - 전체 효과: g = 0.58 [0.42, 0.74]
  - 이질성: I² = 67%

에이전트: E5-민감도분석설계자
실행: 조절변인 분석, 출판 편향
출력:
  - 유의미한 조절변인: 연령집단, 기간, 상호작용 유형
  - 깔때기 그림: 약간의 비대칭 발견

=== 7단계: 원고 준비 ===

[순차 실행]
  1. F2-체크리스트관리자
     실행: PRISMA 2020 준수 확인
     출력: "27/27 항목 완료"

  2. G5-학술스타일감사자
     실행: AI 패턴 검출
     출력: 15개 패턴 검출 (고위험 3개)

  3. G6-학술스타일인간화
     실행: 자연스러운 문체로 변환
     모드: balanced
     출력: "22% 텍스트 수정"

=== 8단계: 저널 투고 ===

에이전트: G1-저널매칭
출력:
  추천 저널:
    1. Educational Psychology Review
       적합도: 0.89, IF: 10.1
    2. British Journal of Educational Technology
       적합도: 0.85, IF: 6.7

===========================================
총 사용 에이전트: 12개
총 체크포인트: 필수 2개, 권장 3개
예상 기간: 4-6개월
===========================================
```

### 예시 2: 신속 정책 연구 (4주)

```yaml
상황: "학교 현장 AI 도구 사용 경험에 대한 교사 인터뷰 분석"
제약: "4주 내 정책 보고서 제출 필요"

=== 1주차: 설계 ===

[병렬 실행]
  ├─ A1-연구질문정제기
  │   출력: "교사들은 AI 교육 도구를 어떻게 인식하고 사용하는가?"
  │
  ├─ A4-연구윤리자문
  │   출력: "면제 가능 - 저위험 면담"
  │
  └─ C2-질적설계컨설턴트
      출력: "일반적 질적 접근법 (시간 제약으로 현상학 제외)"

=== 2주차: 데이터 수집 ===

에이전트: D1-표집전략자문
출력:
  - 전략: 기준 + 편의 표집
  - 기준: "1년 이상 AI 도구 사용 교사"
  - 목표 n: 15명

에이전트: D2-면담전문가
출력:
  - 프로토콜: 반구조화, 45분
  - 질문: 주요 8개 + 추가탐색 12개

=== 3주차: 분석 ===

에이전트: B5-병렬문서처리기
실행: 15개 면담 전사본 병렬 처리
구성: 워커 3개, 배치당 5개
출력: "모든 전사본 추출 및 정리 완료"

에이전트: E2-질적코딩전문가
실행: 신속 주제 분석 (Braun & Clarke 축약)
출력:
  - 코드: 47개
  - 주제: 5개 주요 주제

[병렬 실행]
  ├─ F4-신뢰성검출기
  │   출력: "3개 사례에 대해 구성원 검토 권장"
  │
  └─ A3-악마의대변인
      출력: "대안적 해석: 채택 과정의 확증 편향"

=== 4주차: 보고서 ===

에이전트: G2-학술소통
실행: 정책 보고서 형식
출력: "5페이지 요약 보고서"

에이전트: A6-개념적프레임워크시각화
실행: 주제 지도 생성
출력: "Mermaid 주제 다이어그램"

===========================================
총 사용 에이전트: 10개
기간: 4주
출력물: 정책 보고서 + 기술적 부록
===========================================
```
